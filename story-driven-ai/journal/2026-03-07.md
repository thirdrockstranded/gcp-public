## US-002 Input Flow Revision

After photo upload, the quilt size and block size inputs are dependent on each other — presenting them as independent free-entry fields allows invalid combinations and puts math burden on the user.

**Revised approach:** Sequential input where the second field is constrained by the first.

- If quilt size entered first → only show block sizes that divide evenly into both dimensions
- If block size entered first → constrain or warn on quilt size in real time
- Grid dimensions (columns × rows) display as a live calculation as inputs resolve

**Open questions logged in US-002:**
- Which comes first — quilt size or block size? Needs stakeholder input. Quilters likely have a natural starting point.
- Block size: picker only, or free entry with validation?
