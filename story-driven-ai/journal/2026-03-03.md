## What Was Decided

**The methodology repo is separate from the project repo from day one.** Quiltix is Case Study 1, not the home of the framework. This keeps the methodology portable and reusable across future projects.

**Both repos live under the personal GitHub account** until a formal Green Checkered Pants org account exists. Repo transfers are straightforward when that happens.

**The methodology is solving a different problem than existing frameworks.** LangChain, CrewAI, LangGraph etc. solve runtime agent orchestration. They don't address the specification → generation → verification pipeline. That gap is what this is exploring.

---

## The Core Hypothesis (First Statement)

Structured user stories, stored alongside the code they describe, can serve as a reliable specification layer for AI code generation. The quality of the generated code is measurable against those stories through automated testing. Iterating the story format improves the score.

---

## Story Format Decisions (v0.1)

The initial template includes:
- Epic (one-sentence context)
- Standard As a / I want to / So that format
- Explicit inputs (what data this feature receives)
- Acceptance criteria (each must map to a test)
- Notes (domain knowledge, constraints, deferred items)
- Test coverage declaration (unit + Playwright)

**Open questions not yet resolved:**
- Should stories reference each other explicitly or stay decoupled?
- How granular should acceptance criteria be?
- Should edge cases live in the story or only in the test file?

## Quiltix Domain Knowledge Captured

The session surfaced several quilting-specific constraints that would not be obvious to an LLM without explicit documentation:

- 1/4" seam allowance is the industry default for quilting cotton
- Cut size = finished size + 1/2" (seam allowance on both sides)
- Standard usable fabric width = 42" (pre-washed quilting cotton)
- Yardage always rounds up to nearest 1/8 yard — never down
- These are not arbitrary choices; they reflect decades of quilting convention

This reinforced the value of the Notes section in the story template. Domain knowledge that lives only in a stakeholder's head is a project risk; domain knowledge written into the story file is durable.
